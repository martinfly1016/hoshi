# 星の命式 MVP 交互规格

版本: v0.1  
日期: 2026-05-10  
基准视觉: `site3/DESIGN.md`

## 1. 交互目标

MVP 只追求一个完整闭环：

```text
进入网站
理解产品
开始免费鉴定
填写出生信息
生成命式
阅读基础解释
进入相性/大运/流年等后续入口
```

交互风格可以有仪式感，但每一步都必须清楚、可用、可解释。

## 2. 首页交互

首页第一屏使用 `site3` 的视觉方向。

保留：

- 星图/命式母题。
- 朱印。
- 古典标题气质。
- 安静动效。

需要产品化调整：

- CTA 文案既要有仪式感，也要让用户知道是免费鉴定。
- 首页需要补充一句现代说明，例如“生年月日と出生時間から四柱推命の命式を作成します”。
- 导航项应链接到真实页面，而不是仅做样稿状态切换。

主要动作：

- 点击主 CTA 进入 `/shichusuimei/free/`。
- 点击“四柱推命”进入 `/shichusuimei/`。

## 3. 免费鉴定表单

### 3.1 表单原则

表单可以像“鑑定の儀”，但不能让用户迷失。

每个字段需要同时具备：

- 古典标签。
- 现代说明。
- 输入提示。
- 错误提示。

例如：

```text
生年月日
四柱推命で年柱・月柱・日柱を出すために使います。
```

### 3.2 字段

MVP 字段：

| 字段 | 必填 | 说明 |
|---|---|---|
| 生年月日 | 是 | 年、月、日 |
| 出生時間 | 否 | 支持“不明” |
| 出生地 | 是 | MVP 至少支持日本都道府县 |
| 計算モード | 否 | 默认标准时间；可选真太阳时精细计算 |
| 性别 | 待定 | 只有计算逻辑需要时才必填 |
| 名称 | 否 | 用于结果显示，不参与计算 |

### 3.3 出生时间

用户需要能选择：

- 具体时间。
- 十二时辰。
- 不明。

正式计算建议以具体时间为主，十二时辰可作为视觉化辅助。

出生时间不明时：

- 表单允许继续提交。
- 计算默认使用 12:00 午时。
- 结果页必须把时柱标记为“12:00 午时假设值”，并提示相关解释仅供参考。

23:00-23:59：

- 默认按晚子时/夜子时处理，不跨日。
- 在高级设置中允许切换为 23:00 子初换日。
- 结果页需要显示当前采用的换日规则。

### 3.4 出生地

MVP：

- 都道府县选择。
- 可选城市输入。
- 默认按输入时间直接计算。

精细模式：

- 用户选择真太阳时后，必须填写城市或可解析经纬度的出生地。
- 结果页显示原始输入时间和真太阳时换算后的有效时间。
- 出生地不足时阻止生成精细结果，提示补充出生地。

未来：

- 海外出生。

### 3.5 校验

需要校验：

- 日期不能晚于今天。
- 年份范围合理。
- 必填字段为空时阻止提交。
- 出生时间不明时展示准确性提示。
- 真太阳时模式缺少出生地时阻止提交。

## 4. 提交交互

可以保留 `site3` 的“落印仪式”，但要控制时长。

建议：

- 点击提交后显示 1.2 到 1.8 秒的仪式动画。
- 动画期间显示“命式を作成しています”。
- 计算失败时不能只停在动画，需要清楚错误信息。
- `prefers-reduced-motion` 时跳过落印动画。

## 5. 结果展示

结果页第一屏不应直接给复杂命式表。

推荐顺序：

1. 结果摘要。
2. 用户输入信息确认。
3. 日主和关键标签。
4. 五行平衡。
5. 命式表。
6. 性格。
7. 恋爱。
8. 仕事/才能。
9. 注意说明。
10. 后续入口。

### 5.1 结果摘要

用一段易懂日文说明：

```text
あなたの命式は、〇〇の性質が強く、...
```

避免：

- 绝对断言。
- 恐吓。
- 健康、投资、法律等高风险建议。

### 5.2 命式表

命式表要专业但可读。

需要包含：

- 年柱。
- 月柱。
- 日柱。
- 時柱，如果出生时间不明则提示缺失。
- 天干。
- 地支。
- 五行。
- 通変星。

移动端：

- 不能挤成不可读宽表格。
- 可用 2x2 柱卡片。

### 5.3 五行平衡

建议使用可视化：

- 条形。
- 雷达图。
- 五元素环。

不要求 MVP 做复杂动画。

## 6. 后续入口

结果页底部提供：

- 相性を見る。
- 大運を見る。
- 流年を見る。
- 結果を保存する。
- 結果を共有する。

MVP 中未实现的功能可以显示“準備中”，但不要把它们做成可点击假功能。

## 7. 移动端交互

移动端优先。

要求：

- 首屏 CTA 明确。
- 表单字段单列。
- 输入框字号至少 16px。
- 时辰 chips 不溢出。
- 提交按钮固定或易找到。
- 结果卡片可连续阅读。

`site3/mobile-preview.html` 可作为移动端验收工具。

## 8. Tweaks 面板

`site3` 中 Tweaks 面板适合设计调试。

正式用户版本：

- 默认隐藏。
- 仅通过 `?tweaks=1` 开启。
- 不作为普通用户功能。

## 9. 验收标准

MVP 交互完成标准：

- 用户能从首页进入免费鉴定。
- 用户能填写必要出生信息。
- 表单有校验和错误提示。
- 提交后能看到命式结果。
- 出生时间不明时结果有明确限制说明。
- 移动端流程可完整走完。
- 动效不阻断操作。
- 未实现功能不会伪装成已实现。
