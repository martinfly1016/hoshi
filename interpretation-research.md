# 星の命式 解读功能调研

日期: 2026-05-10  
范围: 四柱推命 / 八字类产品的“结果解读”形态  
目标: 为星の命式后续解释层和结果页产品方案提供参考

## 1. 调研结论

“解读”不是单纯把命式表翻译成文字，而是把计算结果转换为用户能理解、能行动、能继续探索的结果体验。

外部产品大致分成四类：

1. 工具型排盘 + 术语说明。
2. 免费基础解读 + 付费长报告。
3. AI 生成报告 + 可追问对话。
4. 关系 / 恋爱 / 职业等主题化报告。

对星の命式而言，MVP 不建议一开始做超长报告。更合适的形态是：

- 先输出一屏可读的核心摘要。
- 再分模块解释日主、五行、通変星、性格、恋爱、仕事/才能。
- 命式表保留专业信息，但放在摘要之后。
- AI 解读作为“进一步展开 / 追问”能力，而不是替代计算层。

## 2. 竞品观察

### 2.1 AI 报告型

代表：

- 观象八字
- 查八字
- FateSign
- BaziChat
- 四柱推命.AI

共同特点：

- 强调“AI 深度分析报告”。
- 常见模块包括性格、事业、财运、感情、健康、大运流年。
- 会把排盘结果转成可视化图表，例如五行能量、命盘结构、运势时间线。
- 支持追问或模板化分析。

典型形式：

- 观象八字：以“AI 深度分析报告 + 数据可视化 + 智能追问 + PDF 报告”为主要卖点。
- 查八字：免费排盘，付费后提供 20,000 字报告，章节包括八字概要、格局推理、五行喜忌、爱情密码、大运解析、人生周期、流年流月等。
- FateSign：把解读包装成“人生剧本”，按主角设定、大运章节、局势、里程碑、策略来组织，产品感强。
- BaziChat：更像 AI 工作台，提供模板、多人分析、自定义模板、聊天输入和用户档案。
- 四柱推命.AI：注册成本低，强调约 3,000 字 AI 解析、命式图表、保存分享链接。

启发：

- AI 解读最好有固定结构，否则容易变成一段泛泛长文。
- 对普通用户来说，“人格/才能/恋爱/仕事/タイミング”比“格局/用神/神煞”更容易理解。
- 可追问是高价值方向，但要建立在结构化命式数据之上。

### 2.2 免费基础解读 + 付费深度报告

代表：

- あなたの命式
- 查八字
- Four Pillar

共同特点：

- 免费层给用户足够多的即时反馈。
- 付费层卖更长、更细、更个性化的报告。
- 付费前会明确列出“你将获得什么”。

典型形式：

- あなたの命式：免费提供命式、本质性格、十二运、通変星、五行、大运、年运、月别运势、相性、开运物等；深度报告提供大运 10 年详解、未来 3 年月别运势、仕事/恋爱/人际深层分析、开运日历、人生转机。
- Four Pillar：聚焦恋爱场景，一次性购买报告，内容包括 Personality DNA、Love Patterns、Ideal Match Profile、Personal Q&A。
- 查八字：免费排盘，付费生成长报告和二人合盘。

启发：

- 免费解读要能建立信任，不能只给命式表。
- 付费/深度入口不一定一开始做，但 MVP 结果页应该预留“展开更多”的结构。
- 日本市场可以采用“免费基础鑑定 + 詳細鑑定書”语境，比“AI算命报告”更自然。

### 2.3 专业工具 + 白话解释

代表：

- 问真八字
- 准派
- いい運勢ねっと

共同特点：

- 专业排盘仍然是核心。
- 解读服务围绕排盘结果展开。
- 对术语提供解释、判断依据或学习辅助。

典型形式：

- 问真八字：核心是四柱排盘、四柱合盘、神煞、称骨、出生天体图、智能古籍、八字论财等，偏专业工具集合。
- 准派：提出 AI 解读基于排盘结果自动生成，包含提取核心信息、白话说明术语与判断依据、多维趋势、追问互动。
- いい運勢ねっと：强调通変星、十二运星要让专家和初心者都能直观理解；给出“命式表阅读提示”。

启发：

- 对星の命式来说，命式表不能省，但需要“翻译层”。
- 术语解释应该贴在具体结果旁边，不要只做百科页。
- 结果页可以同时服务普通用户和进阶用户：默认看摘要，展开看术语。

### 2.4 日本用户向页面形态

日本四柱推命站点的常见表达：

- 命式
- 本質
- 性格
- 運勢
- 相性
- 大運
- 年運
- 月別運勢
- ラッキーカラー
- 開運アイテム

输入方式常见：

- 生年月日。
- 出生时间可选，或用十二时辰段。
- 性别常作为输入项。
- 有些站点支持“不明”。

结果页常见顺序：

1. 命式。
2. 本質 / 性格。
3. 通変星 / 十二運。
4. 五行バランス。
5. 大運。
6. 年運 / 月別運勢。
7. 相性。
8. 開運建议。

启发：

- 星の命式应优先使用日文用户熟悉的“本質、性格、恋愛、仕事、運勢、相性”。
- “八字”“十神”如果出现，需要配合“四柱推命 / 通変星”的日本语境。
- 第一屏不要直接堆专业表，先给“あなたはどんな命式か”的短摘要。

## 3. 解读功能应拆成三层

### 3.1 计算事实层

来源：`tyme4ts` + 自有 adapter。

输出：

- 四柱。
- 日主。
- 五行。
- 通変星。
- 藏干。
- 计算元信息。
- warning，例如出生时间不明、真太阳时、晚子时规则。

要求：

- 不写主观断语。
- 不生成用户建议。
- 可回归测试。

### 3.2 规则解释层

来源：确定性的映射规则和解释模板。

输出：

- 日主基础性格。
- 五行偏重 / 缺失说明。
- 通変星关键词。
- 时辰不详导致的解释降级说明。
- 当前规则说明。

要求：

- 稳定、可控、可审校。
- 不依赖大模型。
- 适合免费 MVP。

### 3.3 AI 深度解读层

来源：结构化命式数据 + 固定 prompt + 安全边界。

输出：

- 综合摘要。
- 恋爱 / 仕事 / 才能 / 人际 / 运势方向。
- 用户追问。
- 主题报告。

要求：

- 不让模型重新排盘。
- 不允许模型编造计算结果。
- 必须带免责声明和低置信提示。
- 对健康、投资、法律等高风险内容限制为一般性建议。

## 4. MVP 建议形态

第一版结果页建议采用“基础解读卡片 + 专业命式表”的结构。

推荐模块顺序：

1. 一句话摘要  
   例如：あなたの命式は、〇〇の性質が強く、△△を通じて力を発揮しやすいタイプです。

2. 核心标签  
   3 到 5 个短标签，例如“行動力”“調整役”“表現力”“慎重派”。

3. 日主解读  
   解释日主是什么、当前日主的基本倾向。

4. 五行平衡  
   用条形图或五元素卡片展示强弱，并给一段白话说明。

5. 通変星 / 才能倾向  
   选月柱、日支、时柱等重点，不要一次解释所有术语。

6. 恋爱倾向  
   先做轻量版，避免绝对化。

7. 仕事 / 才能倾向  
   以工作风格、适合环境、注意点表达，不直接断职业成败。

8. 命式表  
   专业信息完整展示，给进阶用户确认。

9. 注意说明  
   出生时间不明、真太阳时、晚子时规则等。

10. 后续入口  
   相性、大运、流年、AI 追问、保存分享。

## 5. 免费与后续付费/高级能力边界

MVP 免费层建议包含：

- 命式表。
- 核心摘要。
- 日主。
- 五行平衡。
- 通変星基础说明。
- 性格、恋爱、仕事的短解读。

后续高级层可以包含：

- 大运 10 年周期详解。
- 年运 / 月别运势。
- 相性合盘。
- AI 追问。
- PDF / 图片报告。
- 保存历史档案。

不建议 MVP 就做：

- 20,000 字长报告。
- 全神煞解释。
- 复杂格局 / 用神自动断定。
- 健康、投资、婚姻重大决策的强建议。

## 6. 对星の命式的产品判断

当前最合理路线：

1. 计算层已经暂定固定 `tyme4ts`。
2. 下一步不要直接接 AI 长文，而是先做“规则解释层”。
3. 先让结果页能给稳定、可控、短而准确的基础解读。
4. 再把同一份结构化命式数据交给 AI，生成“更深く読む”或“質問する”。

原因：

- 只靠 AI 容易出现泛化、幻觉和术语误用。
- 只靠表格对普通用户不友好。
- 规则解释层可以建立信任，也能作为 AI prompt 的事实基础。

## 7. 建议下一步

1. 定义 `InterpretationInput`：从计算结果提取解释所需字段。
2. 定义 `InterpretationOutput`：摘要、标签、各模块短文、注意事项。
3. 先写日主、五行、通変星的规则字典。
4. 做一个“纯文本基础解读页”验证信息结构。
5. 再进入视觉化结果页和 AI 深度解读。
